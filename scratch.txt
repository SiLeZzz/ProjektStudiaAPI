GET /permissions/managers:
- zwraca liste EmployeeResponse ograniczona do aktywnych pracownikow z rola Pracownik.
- minimalny fallback dla desktopu: GET /employees?role=Pracownik&isActive=true.

PermissionResponse uses sygnatura as the real key; there is no numeric id:
{
  "sygnatura": "MAG-001",
  "nazwa": "Dostep do magazynu",
  "opis": "...",
  "pracownikZarzadzajacyId": 1,
  "pracownikZarzadzajacy": "Jan Kowalski"
}

CreatePermissionRequest / UpdatePermissionRequest:
- pracownikZarzadzajacyId musi wskazywac aktywnego pracownika z rola Pracownik.

AssignPermissionsRequest:
{
  "sygnaturyUprawnien": ["MAG-001", "FAK-002"],
  "wazneDo": "2026-12-31"
}

Uwaga: wazneDo pozostaje w kontrakcie dla kompatybilnosci, ale przy POST /permissions/employees/{employeeId}/permissions nowe przypisania sa zapisywane jako oczekujace z wazneDo=null. Date waznosci ustawia dopiero pracownik zarzadzajacy przez endpoint approve.

Workflow zatwierdzen:
- GET /permissions/approvals/pending zwraca tylko przypisania z wazneDo=null, gdzie aktualny pracownik jest PracownikZarzadzajacyId uprawnienia.
- POST /permissions/approvals/{employeeId}/{signature}/approve wymaga roli Pracownik.
- approve zwraca 403, gdy aktualny pracownik nie zarzadza wskazanym uprawnieniem.
- approve zwraca 404, gdy przypisanie nie istnieje.
- approve zwraca 409, gdy przypisanie ma juz ustawione wazneDo.
- validUntil jest wymagane i musi byc wieksze lub rowne dacie dzisiejszej.

Error format:
- Validation errors: standard ValidationProblemDetails.
- 401/403/404/409: ProblemDetails with title/detail/status.

Semantics:
- LoginResponse.User.Rola i AuthenticatedUserDto.Rola zwracaja Administrator albo Pracownik.
- Zwykly Pracownik nie ma dostepu do endpointow administracyjnych.
- DELETE /employees/{id} deactivates employee by setting czyAktywny=false.
- DELETE /positions/{id} returns 409 if employees reference the position.
- DELETE /companies/{id}, /departments/{id}, /permissions/{sygnatura} return 409 when dependencies block deletion.
- Bootstrap tworzy admina oraz testowego managera permission.manager z rola Pracownik i haslem bootstrapowym; domyslne uprawnienia testowe wskazuja tego managera.
